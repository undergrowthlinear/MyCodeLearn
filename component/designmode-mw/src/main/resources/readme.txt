设计模式
分为三类:
Create型(5)----SBPFF
			S----单例模式----创建唯一对象,提供一个全局访问点
			B----生成器模式----封装对象的创建过程,允许按照步骤创建对象
			P----原型模式----创建对象昂贵或者复杂，使用原型模式
			F----工厂方法----由子类决定创建哪个对象
			F----抽象工厂----创建依赖的家族对象，而无需指定特定的类
Structure型(7)----ABCDPFF
			A----适配器模式----封装对象,提供不同的访问接口
			B----桥接模式----独立改变对象和抽象
			C----组合模式----用一致的方式处理集合与集合元素
			D----装饰模式----封装对象，提供额外的行为
			P----代理模式----封装对象，提供对对象的访问权限
			F----外观模式----简化接口
			F----蝇量模式----单个实例提供不同的虚拟实例
Behavior型(11)----IICCSSMMMVO
			I----迭代器模式----迭代集合，无需暴露集合内部
			I----解释器模式----为语言创建解释
			C----命令模式----将请求封装成对象
			C----责任链模式----为请求创建对象链
			S----策略模式----将可变的行为封装起来,使用委托决定哪一个
			S----状态模式----将可变的状态封装起来,使用委托决定使用哪个
			M----模板方法----由子类决定实现算法的哪个步骤
			M----备忘录模式----返回对象之前的状态
			M----中介者模式----沟通复杂对象之间的连接
			V----访问者模式----为组合添加行为
			O----观察者模式----在状态改变时对象能够被通知到
